# P2L2 Efinix RISC-V Kit CMake Project Template

## Customizations and adaptations

Some information is not automatically taken from the generated Efinix BSP, the following table specifies tunables and where they can be set:

| Tuneable      | Default Value | File              | Description                                                                                                                                                       |
| ------------- | ------------- | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RISCV_ISA     | "rv32imafdc"  | CMakePresets.json | Selects the capabilities of the CPU. Default is for hardened SoC, may need modifications for softcore                                                             |
| RISCV_ABI     | "ilp32d"      | CMakePresets.json | Sets the calling convention. DEfault is 32-bit with double-precision hard-float. Modify for CPU capabilities and if linking to pre-compiled lib with specific ABI |
| SMP           | True          | CMakePresets.json | Set to false for single-core configurations. Note: leave true if you want to run single-core on a multicore CPU.                                                  |
| NOSTD         | False         | CMakeLists.txt    | Set to true to exclude C Standard libraries for size optimization.                                                                                                |
| mcmodel       | medlow        | CMakeLists.txt    | Uncomment the correct line to select the code generation mode for addrresses. See comments in CMakeLists.txt                                                      |
| LINKER_SCRIPT | "ld/dram.ld"  | CMakeLists.txt    | Configures which memory the application is linked into. Comes with two default linkerscripts: dram.ld and sram.ld                                                 |

CMakeUserPresets.json is generated by the VSCode extension. Alternatively, copy the template and adapt the toolchain paths
